# Download HDMA fetal multiome dataset (Liu et al., Nature 2026)

## Context

The paper at https://www.nature.com/articles/s41586-026-10326-9 — Liu, Jessa, Kim, Ng et al., **"Multiomics and deep learning dissect regulatory syntax in human development"** (Greenleaf/Kundaje/Farh, Nature 2026) — releases the **Human Development Multiomic Atlas (HDMA)**: 817,740 fetal cells across **12 organs** (Adrenal, Brain, Eye, Heart, Liver, Lung, Muscle, Skin, Spleen, StomachEsophagus, Thymus, Thyroid), 203 cell types, post‑conception weeks 10–23, profiled with **SHARE‑seq** (paired RNA + ATAC).

User answers to clarifying questions:
- Want **cell-level fragments + RNA mtx + per-cell metadata + global caCRE + ChromBPNet training regions** for **all 12 organs**.
- Storage on **Crick** under `/nemo/lab/briscoej/home/users/kleshcv/large_data/HDMA/` (directory already exists, currently empty).
- Goal: integrate into the internal whole-embryo workflow (4-modality multimodal training pattern). Cluster-level CRE benchmark deferred to a separate plan.

### Direct answers to the user's two questions
1. **Cell-level fragments — YES.** One tabix-indexed `<sample>.fragments.tsv.gz` + `.tbi` per sample on Zenodo, with cell barcodes embedded (SHARE-seq `CL131_C01+A01+A01` format). Same shape as the immune-integration GSE-based loaders.
2. **Cluster-level peak coordinates — YES.** `hdma_global_caCREs.bed` (~37 MB, ~1 M global CREs) + per-cluster `all_training_regions.tar.gz` (~1.7 GB, ChromBPNet peak/non-peak training regions per cluster) on Zenodo record 17427146. Per-organ ArchR projects (separate, larger records) hold per-cluster PeakMatrix and peak BEDs but are out of scope for this plan.

### Why no Seurat RDS / R dependency
Per-cell obs is **already deposited as a standalone CSV** — `per_cell_meta.csv` (37 MB) on Zenodo 17427146 — so we do **not** need to read `<organ>_RNA_obj_clustered_final.rds` (the Lung Seurat alone is 38 GB). Existing immune-integration loaders never call `readRDS` either; they only handle Seurat-style barcode suffixes inside CSV annotations (`data_loading_utils.py:871,1220`). HDMA can stay R-free.

## Implementation steps

### 1. Download driver script — new

Create [scripts/zenodo_download/](scripts/zenodo_download/) mirroring [scripts/geo_download/](scripts/geo_download/).

- [scripts/zenodo_download/zenodo_records.tsv](scripts/zenodo_download/zenodo_records.tsv) — TSV with columns `dataset_type`, `organ`, `record_id`, `doi` (rows = the 14 records above).
- [scripts/zenodo_download/get_urls.py](scripts/zenodo_download/get_urls.py) — port of Greenleaf's helper (`/tmp/HDMA/DATA.md` lines 50–138). Reads record IDs from the TSV, queries `https://zenodo.org/api/records/{id}`, writes `urls_file.txt` (one direct download URL per file). Use `requests`; retries on transient 5xx. Default landing dir: argv-controlled.
- [scripts/zenodo_download/filter_urls.sh](scripts/zenodo_download/filter_urls.sh) — `grep '\.fragments\.tsv\.gz' urls_file.txt > urls_fragments.txt`, etc., per the DATA.md cookbook (`/tmp/HDMA/DATA.md:211-224`).
- [scripts/zenodo_download/download_hdma.sh](scripts/zenodo_download/download_hdma.sh) — orchestrator: `cd $DEST && wget -c -i urls_fragments.txt && wget -c -i urls_rna_mtx.txt && wget -c -i urls_metadata.txt`. Resumable (`-c`). Run from a long-lived bsub/Slurm job (no GPU needed); recommended `tmux` or `nohup` for the parent shell.

Subagents lacking Write/Edit must NOT use heredocs to create these — per the project subagent rule. Files are written via the Write tool from the parent.

### 2. Sample mapping CSV — new helper

[scripts/zenodo_download/build_sample_mapping.py](scripts/zenodo_download/build_sample_mapping.py): scan `rna/` and `atac/` directories, parse sample_id `T(\d+)_b(\d+)_(<organ>)_PCW(\d+)` regex, emit `manifest/hdma_sample_mapping.csv` with columns:

```
sample_id, organ, organ_code, donor_id, batch, PCW, barcodes_path, features_path, matrix_path, fragment_file_path
```

Schema mirrors [docs/notebooks/immune_integration/data_loading_utils.py](docs/notebooks/immune_integration/data_loading_utils.py) per-dataset `sample_mapping.csv` (e.g. `load_crohns_pbmc_gse244831` at line 1098 expects `gex_h5_path` / `fragment_file_path`).

### 3. Per-organ rename pass — new

Filenames on Zenodo are `<sample>.barcodes.tsv.gz` etc., but the immune-integration loaders expect Cell Ranger-style `barcodes.tsv.gz` inside a sample directory. A short Python helper [scripts/zenodo_download/reorganize_files.py](scripts/zenodo_download/reorganize_files.py) moves each downloaded file into `rna/<sample_id>/<role>.tsv.gz` or `atac/<sample_id>/<role>` (using `shutil.move`, idempotent). Single pass, no decompression — matches the immune pattern of keeping fragments tabix-indexed and unmodified.

### 4. HDMA loader — new module

[docs/notebooks/whole_embryo_integration/data_loading_utils.py](docs/notebooks/whole_embryo_integration/data_loading_utils.py) (new directory mirroring `immune_integration/`):

- `load_hdma_organ(organ, data_root, per_cell_meta_path) -> AnnData` — reads `manifest/hdma_sample_mapping.csv`, filters to the organ, loops samples calling `scanpy.read_mtx`, sets `var_names` from features, `obs_names` to `<sample_id>#<barcode>` to match `per_cell_meta.csv` keys. Concatenates with `anndata.concat(..., join='outer')`. Joins per-cell metadata. Returns AnnData with `obs.fragment_file_path` populated per cell (carried from sample mapping). Pattern from [docs/notebooks/immune_integration/data_loading_utils.py:1098](docs/notebooks/immune_integration/data_loading_utils.py#L1098) (`load_crohns_pbmc_gse244831`).
- `load_hdma_all(data_root, per_cell_meta_path) -> AnnData` — concatenates all 12 organs.

No R dependency. No Seurat readers.

### 5. Notebook stub — defer beyond download

A `hdma_data_loading.ipynb` analogous to `bm_pbmc_data_loading.ipynb` is **not in scope** for this plan; it will be the first downstream task once the data is on disk.
